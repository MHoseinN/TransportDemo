# Mermaid User Flows — سامانه ترابری و احکام مأموریت

## 1) User Flow اصلی سامانه

```mermaid
flowchart TD
    A([شروع]) --> B{پیش‌نیازهای سامانه آماده است؟}
    B -- خیر --> C[ثبت شعب]
    C --> D[ثبت شهر و استان]
    D --> E[ثبت خودرو]
    E --> F[ثبت راننده]
    F --> G[ثبت قرارداد]
    G --> H[درخواست مأموریت]

    B -- بله --> H

    H --> I[ارسال در گردش تأیید سازمانی]
    I --> J{همه سطوح تأیید کردند؟}
    J -- خیر --> K([رد درخواست و خاتمه])
    J -- بله --> L[ورود به کارتابل ترابری]

    L --> M[انتخاب راننده و انتصاب خودرو]
    M --> N{راننده و خودرو معتبر و در دسترس هستند؟}
    N -- خیر --> M
    N -- بله --> O[انجام مأموریت]

    O --> P{مأموریت درون‌شهری ساعتی است؟}
    P -- بله --> Q[ثبت ورود و خروج راننده]
    P -- خیر --> R[ثبت اطلاعات و کیلومتر مأموریت]

    Q --> R
    R --> S[محاسبه حقوق و هزینه مأموریت]

    S --> T{سازمان مدل محاسبات خاص دارد؟}
    T -- بله --> U[اعمال قواعد محاسبات سایر سازمان‌ها]
    T -- خیر --> V[ثبت نهایی و گزارش‌گیری]

    U --> V
    V --> W([پایان])
```

## 2) فرآیند ثبت شهر و استان

```mermaid
flowchart TD
    A([شروع]) --> B[ورود به فرم ثبت شهر و استان]
    B --> C[انتخاب یا ثبت استان]
    C --> D[ثبت شهر وابسته به استان]
    D --> E[اعتبارسنجی: تکمیل بودن و عدم تکرار]
    E --> F{اطلاعات معتبر است؟}
    F -- خیر --> G[نمایش خطا / اصلاح اطلاعات]
    G --> C
    F -- بله --> H[ذخیره رکورد]
    H --> I[قابل استفاده شدن در درخواست مأموریت]
    I --> J([پایان])
```

## 3) فرآیند ثبت خودرو

```mermaid
flowchart TD
    A([شروع]) --> B[ورود به فرم ثبت خودرو]
    B --> C[انتخاب نوع خودرو و وضعیت شخصی/سازمانی]
    C --> D[ورود شماره شاسی، پلاک، مدل و رنگ]
    D --> E[ورود تاریخ بیمه شخص ثالث و بدنه]
    E --> F{فیلدهای اصلی کامل و معتبرند؟}
    F -- خیر --> G[نمایش اخطار و بازگشت برای اصلاح]
    G --> C
    F -- بله --> H{بیمه معتبر است؟}
    H -- خیر --> I[نمایش هشدار بیمه]
    I --> J{با وجود هشدار، مجاز به ثبت است؟}
    J -- خیر --> C
    J -- بله --> K[ثبت خودرو]
    H -- بله --> K
    K --> L[ایجاد رکورد جستجوپذیر خودرو]
    L --> M([پایان])
```

## 4) فرآیند ثبت راننده

```mermaid
flowchart TD
    A([شروع]) --> B[ورود به فرم ثبت راننده]
    B --> C[انتخاب شعبه و نمایندگی]
    C --> D[ورود اطلاعات هویتی]
    D --> E[ورود شماره موبایل و شماره گواهینامه]
    E --> F[ورود اطلاعات بیمه]
    F --> G{اطلاعات اصلی کامل و معتبر است؟}
    G -- خیر --> H[نمایش خطا / اصلاح اطلاعات]
    H --> C
    G -- بله --> I{بیمه معتبر است؟}
    I -- خیر --> J[نمایش هشدار بیمه]
    J --> K{ثبت ادامه یابد؟}
    K -- خیر --> C
    K -- بله --> L[ثبت راننده]
    I -- بله --> L
    L --> M[ایجاد رکورد جستجوپذیر راننده]
    M --> N([پایان])
```

## 5) فرآیند ثبت قرارداد

```mermaid
flowchart TD
    A([شروع]) --> B[ورود به فرم ثبت قرارداد]
    B --> C[انتخاب راننده / طرف قرارداد]
    C --> D[انتخاب خودرو]
    D --> E[ثبت تاریخ شروع و پایان]
    E --> F[ثبت مبالغ: ساعتی، رفت، برگشت، خواب]
    F --> G[ثبت وضعیت بیمه قرارداد]
    G --> H{اطلاعات کامل و معتبر است؟}
    H -- خیر --> I[نمایش خطا / اصلاح اطلاعات]
    I --> C
    H -- بله --> J[ثبت قرارداد]
    J --> K[قرارداد آماده استفاده در محاسبات ماهانه]
    K --> L([پایان])
```

## 6) فرآیند درخواست مأموریت

```mermaid
flowchart TD
    A([شروع]) --> B[ورود به فرم درخواست مأموریت]
    B --> C[انتخاب نوع رفت و برگشت: زمینی / هوایی]
    C --> D[انتخاب نوع مأموریت: ساعتی / روزانه]
    D --> E[ثبت زمان شروع و پایان]
    E --> F[ثبت مبدا و مقصد: استان، شهر، آدرس]
    F --> G[ثبت علت مأموریت و توضیحات]
    G --> H{همراه/همراهان دارد؟}
    H -- بله --> I[انتخاب سایر مأمورین]
    H -- خیر --> J[ثبت درخواست]
    I --> J

    J --> K[ارسال به گردش تأیید]
    K --> L{سطح فعلی تأیید می‌کند؟}
    L -- خیر --> M([رد درخواست و خاتمه])
    L -- بله --> N{تأییدکننده دیگری باقی مانده؟}
    N -- بله --> K
    N -- خیر --> O[ارسال به کارتابل ترابری]
    O --> P([پایان])
```

## 7) فرآیند ثبت ورود و خروج راننده

```mermaid
flowchart TD
    A([شروع]) --> B[ورود به فرم ثبت ورود و خروج]
    B --> C[انتخاب راننده]
    C --> D[انتخاب تاریخ تردد]
    D --> E[ثبت ساعت ورود]
    E --> F[ثبت ساعت خروج]
    F --> G{برای همان روز رکورد/رکوردهای قبلی وجود دارد؟}
    G -- بله --> H[بررسی ثبت چند تردد یا ویرایش رکورد]
    G -- خیر --> I[ذخیره رکورد]

    H --> J{کاربر مجاز به اصلاح/افزودن است؟}
    J -- خیر --> K([خاتمه بدون ثبت])
    J -- بله --> I

    I --> L[ثبت تردد برای محاسبات ماهانه]
    L --> M([پایان])
```

## 8) فرآیند انتخاب راننده و انتصاب خودرو به مأموریت

```mermaid
flowchart TD
    A([شروع]) --> B[ورود به کارتابل ترابری]
    B --> C[انتخاب مأموریت تأییدشده]
    C --> D[نمایش لیست رانندگان آماده سرویس‌دهی]
    D --> E[نمایش لیست خودروهای آماده سرویس‌دهی]
    E --> F[انتخاب راننده]
    F --> G[انتخاب خودرو]
    G --> H{راننده در دسترس است؟}
    H -- خیر --> D
    H -- بله --> I{خودرو معتبر، سالم و بیمه‌دار است؟}
    I -- خیر --> E
    I -- بله --> J[نمایش پیام تأیید انتصاب]
    J --> K{کاربر تأیید می‌کند؟}
    K -- خیر --> F
    K -- بله --> L[ثبت انتصاب راننده و خودرو]
    L --> M[ایجاد رکورد مأموریت تخصیص‌یافته]
    M --> N([پایان])
```

## 9) فرآیند ثبت اطلاعات و کیلومتر مأموریت

```mermaid
flowchart TD
    A([شروع]) --> B[ورود به فرم ثبت اطلاعات مأموریت]
    B --> C[انتخاب مأموریت تخصیص‌یافته]
    C --> D[ثبت تاریخ و ساعت رفت]
    D --> E[ثبت تاریخ و ساعت برگشت]
    E --> F[ثبت کیلومتر رفت و برگشت]
    F --> G[محاسبه مسافت طی‌شده]
    G --> H[ورود هزینه‌ها: کیلومتری، خواب، توقفی، جریمه، اضافه]
    H --> I{اطلاعات کامل و معتبر است؟}
    I -- خیر --> J[نمایش خطا / اصلاح اطلاعات]
    J --> D
    I -- بله --> K[محاسبه حقوق مأموریت]
    K --> L{نیاز به محاسبه مجدد دارد؟}
    L -- بله --> H
    L -- خیر --> M[ثبت نهایی اطلاعات مأموریت]
    M --> N[آماده برای گزارش مالی و عملیاتی]
    N --> O([پایان])
```

## 10) فرآیند محاسبات سایر سازمان‌ها (الگوی توسعه‌پذیر)

```mermaid
flowchart TD
    A([شروع]) --> B[انتخاب سازمان / مدل محاسبات]
    B --> C{نوع محاسبه چیست؟}

    C -- درون‌شهری ساعتی --> D[خواندن ساعات ورود و خروج]
    D --> E[ضرب در مبلغ ساعتی قرارداد]
    E --> Z[تولید مبلغ قابل پرداخت]

    C -- درون‌شهری کیلومتری --> F[خواندن کیلومتر مأموریت]
    F --> G[ضرب در نرخ کیلومتری]
    G --> Z

    C -- برون‌شهری --> H[خواندن کیلومتر رفت و برگشت]
    H --> I[محاسبه مبلغ کیلومتری]
    I --> J[محاسبه ساعات رانندگی]
    J --> K[محاسبه ساعات خواب]
    K --> L[ضرب ساعات خواب در نرخ خواب]
    L --> M[اعمال توقفی / جریمه / اضافات]
    M --> Z

    Z --> N[ثبت در کارکرد و حقوق راننده]
    N --> O([پایان])
```
