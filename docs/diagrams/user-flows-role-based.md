# Mermaid Role-Based User Flows — سامانه ترابری و احکام مأموریت

> نکته: در Mermaid، نمایش lane رسمی مثل BPMN وجود ندارد؛ برای نمایش نقش‌ها از `subgraph` به‌عنوان معادل swimlane استفاده شده است.

---

## 1) User Flow اصلی سامانه — Role Based

```mermaid
flowchart LR

    subgraph U1[کاربر / درخواست‌دهنده]
        A([شروع])
        H[ثبت درخواست مأموریت]
    end

    subgraph M1[مدیران / سطوح تأیید]
        I[بررسی در گردش تأیید]
        J{همه سطوح تأیید کردند؟}
        K([رد درخواست و خاتمه])
    end

    subgraph T1[کارشناس / مسئول ترابری]
        B{پیش‌نیازها آماده است؟}
        C[ثبت شعب]
        D[ثبت شهر و استان]
        E[ثبت خودرو]
        F[ثبت راننده]
        G[ثبت قرارداد]
        L[دریافت در کارتابل ترابری]
        M[انتخاب راننده و انتصاب خودرو]
        N{راننده و خودرو معتبر و در دسترس هستند؟}
        O[اعزام و انجام مأموریت]
        Q[ثبت ورود و خروج راننده]
        R[ثبت اطلاعات و کیلومتر مأموریت]
    end

    subgraph F1[مالی / محاسبات]
        S[محاسبه حقوق و هزینه مأموریت]
        T{مدل محاسبات خاص سازمان فعال است؟}
        U[اعمال قواعد محاسبات سایر سازمان‌ها]
        V[ثبت نهایی و گزارش‌گیری]
        W([پایان])
    end

    A --> B
    B -- خیر --> C --> D --> E --> F --> G --> H
    B -- بله --> H
    H --> I --> J
    J -- خیر --> K
    J -- بله --> L --> M --> N
    N -- خیر --> M
    N -- بله --> O
    O --> P{نوع مأموریت}
    P -- درون‌شهری ساعتی --> Q --> R
    P -- سایر --> R
    R --> S --> T
    T -- بله --> U --> V --> W
    T -- خیر --> V --> W
```

## 2) فرآیند ثبت شهر و استان — Role Based

```mermaid
flowchart LR

    subgraph R1[درخواست‌دهنده / کاربر]
        A([شروع])
        B[ورود به فرم شهر و استان]
        C[انتخاب یا ثبت استان]
        D[ثبت شهر وابسته به استان]
    end

    subgraph IT1[واحد فناوری اطلاعات / مدیر داده پایه]
        E[اعتبارسنجی داده]
        F{اطلاعات معتبر و غیرتکراری است؟}
        G[نمایش خطا / اصلاح اطلاعات]
        H[ذخیره رکورد]
        I[قابل استفاده شدن در درخواست مأموریت]
        J([پایان])
    end

    A --> B --> C --> D --> E --> F
    F -- خیر --> G --> C
    F -- بله --> H --> I --> J
```

## 3) فرآیند ثبت خودرو — Role Based

```mermaid
flowchart LR

    subgraph A1[کاربر / مسئول اداری]
        A([شروع])
        B[ورود به فرم ثبت خودرو]
        C[انتخاب نوع خودرو و وضعیت شخصی/سازمانی]
        D[ورود شماره شاسی، پلاک، مدل و رنگ]
        E[ورود تاریخ بیمه شخص ثالث و بدنه]
    end

    subgraph A2[مسئول اداری / پشتیبانی]
        F{فیلدهای اصلی کامل و معتبرند؟}
        G[نمایش اخطار و اصلاح]
        H{بیمه معتبر است؟}
        I[نمایش هشدار بیمه]
        J{با وجود هشدار، ثبت مجاز است؟}
        K[ثبت خودرو]
        L[ایجاد رکورد جستجوپذیر]
        M([پایان])
    end

    A --> B --> C --> D --> E --> F
    F -- خیر --> G --> C
    F -- بله --> H
    H -- خیر --> I --> J
    J -- خیر --> C
    J -- بله --> K
    H -- بله --> K
    K --> L --> M
```

## 4) فرآیند ثبت راننده — Role Based

```mermaid
flowchart LR

    subgraph D1[کاربر / ثبت‌کننده اطلاعات]
        A([شروع])
        B[ورود به فرم ثبت راننده]
        C[انتخاب شعبه و نمایندگی]
        D[ورود اطلاعات هویتی]
        E[ورود شماره موبایل و گواهینامه]
        F[ورود اطلاعات بیمه]
    end

    subgraph D2[مسئول اداری / پشتیبانی]
        G{اطلاعات اصلی کامل و معتبر است؟}
        H[نمایش خطا / اصلاح اطلاعات]
        I{بیمه معتبر است؟}
        J[نمایش هشدار بیمه]
        K{ثبت ادامه یابد؟}
        L[ثبت راننده]
    end

    subgraph D3[مالی / بیمه]
        M[استفاده از اطلاعات برای کنترل و محاسبات]
        N([پایان])
    end

    A --> B --> C --> D --> E --> F --> G
    G -- خیر --> H --> C
    G -- بله --> I
    I -- خیر --> J --> K
    K -- خیر --> C
    K -- بله --> L
    I -- بله --> L
    L --> M --> N
```

## 6) فرآیند درخواست مأموریت — Role Based

```mermaid
flowchart LR

    subgraph M2[درخواست‌دهنده مأموریت]
        A([شروع])
        B[ورود به فرم درخواست مأموریت]
        C[انتخاب نوع رفت و برگشت]
        D[انتخاب نوع مأموریت]
        E[ثبت زمان شروع و پایان]
        F[ثبت مبدا و مقصد]
        G[ثبت علت مأموریت و توضیحات]
        H{همراه/همراهان دارد؟}
        I[انتخاب سایر مأمورین]
        J[ثبت درخواست]
    end

    subgraph M3[مدیر / سطوح تأیید]
        K[بررسی درخواست در گردش]
        L{سطح فعلی تأیید می‌کند؟}
        M([رد درخواست و خاتمه])
        N{تأییدکننده دیگری باقی مانده؟}
    end

    subgraph M4[واحد ترابری]
        O[دریافت در کارتابل ترابری]
        P([پایان])
    end

    A --> B --> C --> D --> E --> F --> G --> H
    H -- بله --> I --> J
    H -- خیر --> J
    J --> K --> L
    L -- خیر --> M
    L -- بله --> N
    N -- بله --> K
    N -- خیر --> O --> P
```

        L[ذخیره رکورد]
        M[استفاده در محاسبات ماهانه]
        N([پایان])
    end

    A --> B --> C --> D --> E --> F --> G --> H
    H -- بله --> I --> J
    J -- خیر --> K
    J -- بله --> L
    H -- خیر --> L
    L --> M --> N

## 8) فرآیند انتخاب راننده و انتصاب خودرو به مأموریت — Role Based

```mermaid
flowchart LR

    subgraph T2[کارشناس / مسئول ترابری]
        A([شروع])
        B[ورود به کارتابل ترابری]
        C[انتخاب مأموریت تأییدشده]
        D[نمایش لیست رانندگان آماده سرویس‌دهی]
        E[نمایش لیست خودروهای آماده سرویس‌دهی]
        F[انتخاب راننده]
        G[انتخاب خودرو]
        J[نمایش پیام تأیید انتصاب]
        K{تأیید انتصاب؟}
        L[ثبت انتصاب راننده و خودرو]
    end

    subgraph T3[کنترل منابع / اعتبارسنجی]
        H{راننده در دسترس است؟}
        I{خودرو معتبر، سالم و بیمه‌دار است؟}
    end

    subgraph T4[راننده]
        M[دریافت مأموریت تخصیص‌یافته]
        N([پایان])
    end

    A --> B --> C --> D --> E --> F --> G --> H
    H -- خیر --> D
    H -- بله --> I
    I -- خیر --> E
    I -- بله --> J --> K
    K -- خیر --> F
    K -- بله --> L --> M --> N
```

## 9) فرآیند ثبت اطلاعات و کیلومتر مأموریت — Role Based

```mermaid
flowchart LR

    subgraph I1[راننده]
        A([شروع])
        B[ارائه زمان رفت و برگشت]
        C[ارائه کیلومتر رفت و برگشت]
    end

    subgraph I2[کاربر / مسئول ثبت]
        D[ورود به فرم ثبت اطلاعات مأموریت]
        E[انتخاب مأموریت تخصیص‌یافته]
        F[ثبت تاریخ و ساعت رفت]
        G[ثبت تاریخ و ساعت برگشت]
        H[ثبت کیلومتر رفت و برگشت]
        I[محاسبه مسافت طی‌شده]
        J[ورود هزینه‌ها: کیلومتری، خواب، توقفی، جریمه، اضافه]
        K{اطلاعات کامل و معتبر است؟}
        L[نمایش خطا / اصلاح]
    end

    subgraph I3[مالی]
        M[محاسبه حقوق مأموریت]
        N{نیاز به محاسبه مجدد دارد؟}
        O[ثبت نهایی اطلاعات مأموریت]
        P[گزارش مالی و عملیاتی]
        Q([پایان])
    end

    A --> B --> C --> D --> E --> F --> G --> H --> I --> J --> K
    K -- خیر --> L --> F
    K -- بله --> M --> N
    N -- بله --> J
    N -- خیر --> O --> P --> Q
```

## 10) فرآیند محاسبات سایر سازمان‌ها — Role Based

```mermaid
flowchart LR

    subgraph X1[کارشناس ترابری / ثبت‌کننده]
        A([شروع])
        B[انتخاب سازمان / مدل محاسبات]
        C{نوع محاسبه}
        D[خواندن ساعات ورود و خروج]
        F[خواندن کیلومتر مأموریت]
        H[خواندن کیلومتر رفت و برگشت]
    end

    subgraph X2[واحد مالی]
        E[ضرب در مبلغ ساعتی قرارداد]
        G[ضرب در نرخ کیلومتری]
        I[محاسبه مبلغ کیلومتری]
        J[محاسبه ساعات رانندگی]
        K[محاسبه ساعات خواب]
        L[ضرب ساعات خواب در نرخ خواب]
        M[اعمال توقفی / جریمه / اضافات]
        Z[تولید مبلغ قابل پرداخت]
        N[ثبت در کارکرد و حقوق راننده]
        O([پایان])
    end

    A --> B --> C
    C -- درون‌شهری ساعتی --> D --> E --> Z
    C -- درون‌شهری کیلومتری --> F --> G --> Z
    C -- برون‌شهری --> H --> I --> J --> K --> L --> M --> Z
    Z --> N --> O
```
