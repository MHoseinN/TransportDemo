# Definition of Ready / Done

## Definition of Ready
A card can move to Ready for Development only when:
1. Goal is clear.
2. User role is clear.
3. Dependencies are listed.
4. API requirements are clear.
5. UI requirements are clear.
6. Acceptance Criteria are written.
7. Input/output data are clear.
8. Error states are clear.
9. Permissions are clear.

## Definition of Done
A card can move to Done only when:
1. Code is implemented.
2. Code Review is completed.
3. Acceptance Criteria are passed.
4. Permissions are enforced.
5. Main errors are handled.
6. API is documented in Swagger/OpenAPI, for backend cards.
7. UI has loading/error/empty states, for frontend cards.
8. Audit or read log is implemented where required.
9. PO or product owner accepts the result.
