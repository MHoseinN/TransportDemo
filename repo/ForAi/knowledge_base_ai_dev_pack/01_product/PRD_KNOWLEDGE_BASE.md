# PRD — ماژول پایگاه دانش

## 1. معرفی
ماژول پایگاه دانش یک سرویس مستقل تحت وب است که برای مدیریت، انتشار، جستجو، مشاهده، نسخه‌بندی، تأیید و ثبت مطالعه اسناد دانشی سازمان آموزشی طراحی می‌شود.

## 2. هدف محصول
هدف اصلی، ایجاد یک منبع واحد و قابل اعتماد برای آیین‌نامه‌ها، بخشنامه‌ها، روال‌ها و پیام‌های روزانه است تا کاربران و اپراتورها بتوانند سریع، دقیق و بر اساس آخرین اطلاعات معتبر پاسخ دهند.

## 3. کاربران اصلی
- مدیر / ادمین ارشد
- سرپرست
- اپراتور / کاربر مشاهده‌کننده
- نرم‌افزار اصلی به‌عنوان مصرف‌کننده API

## 4. انواع محتوا
### 4.1 اسناد رسمی
- آیین‌نامه
- بخشنامه
- روال

این سه نوع از نظر فرآیند مشابه هستند: ثبت، تأیید، انتشار، ویرایش، نسخه‌بندی و آرشیو.

### 4.2 پیام روزانه
پیام روزانه برای همه کاربران قابل مشاهده است و نیاز به تأیید مدیر ندارد. تاریخ و ساعت ارسال پیام باید ذخیره شود.

## 5. دامنه MVP
### داخل دامنه
- مدیریت دسته‌بندی‌ها: معاونت، بخش، موضوع، زیرموضوع
- ثبت سند رسمی توسط مدیر و سرپرست
- تأیید/رد اسناد رسمی ثبت‌شده توسط سرپرست
- ویرایش سند
- افزودن نسخه جدید
- آرشیو نسخه قبلی پس از انتشار نسخه جدید
- مشاهده اسناد جاری و قدیمی
- هشدار واضح برای اسناد قدیمی
- جستجو و فیلتر اسناد
- ثبت پیام روزانه
- ثبت مطالعه شد
- API برای نرم‌افزار اصلی

### خارج از دامنه فعلی
- ثبت تماس
- مدیریت صف تماس
- اتصال مستقیم به نرم‌افزار تلفنی
- پیشنهاد هوشمند سند با AI
- جستجوی کامل داخل PDF/Word آپلودی
- OCR تصاویر و فایل‌های اسکن‌شده

## 6. قواعد کلیدی محصول
- مدیر می‌تواند سند رسمی را مستقیم منتشر کند.
- سرپرست می‌تواند سند رسمی ثبت کند، اما انتشار آن نیازمند تأیید مدیر است.
- ویرایش سند توسط سرپرست نیازمند تأیید مدیر است.
- ویرایش عادی سند نسخه جدید محسوب نمی‌شود.
- نسخه جدید فقط با عملیات «افزودن نسخه جدید» ایجاد می‌شود.
- پس از انتشار نسخه جدید، نسخه قبلی حذف نمی‌شود و با وضعیت قدیمی/آرشیوی باقی می‌ماند.
- همه نقش‌ها می‌توانند اسناد قدیمی را مشاهده کنند، اما باید هشدار قدیمی بودن نمایش داده شود.
- پیام روزانه نیازمند تأیید نیست و برای همه نقش‌ها قابل مشاهده است.
- دکمه «مطالعه شد» برای اسناد جاری و قدیمی وجود دارد.

## 7. معیار موفقیت MVP
- کاربران بتوانند سند را سریع جستجو و مشاهده کنند.
- اسناد تأییدنشده برای کاربران عادی دیده نشوند.
- نسخه جاری و نسخه قدیمی از هم قابل تشخیص باشند.
- مدیر بتواند درخواست‌های سرپرست را تأیید یا رد کند.
- نرم‌افزار اصلی بتواند از API سند بگیرد.
