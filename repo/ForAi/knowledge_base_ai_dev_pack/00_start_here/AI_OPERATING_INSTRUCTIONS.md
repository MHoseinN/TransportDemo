# AI Operating Instructions

این فایل برای ابزار AI Developer / Codex نوشته شده است.

## نقش تو
تو توسعه‌دهنده بک‌اند و فرانت‌اند هستی که باید بر اساس اسناد این فولدر، MVP ماژول پایگاه دانش را پیاده‌سازی کنی.

## قوانین کار
1. قبل از کدنویسی، `README_FOR_AI.md`، `PRD_KNOWLEDGE_BASE.md`، `BUSINESS_RULES.md`، `ROLES_AND_PERMISSIONS.md`، `DOMAIN_MODEL.md` و `openapi.yaml` را بخوان.
2. از قوانین محصولی موجود تخطی نکن.
3. اگر تناقض دیدی، فایل `OPEN_QUESTIONS.md` را به‌روزرسانی کن و قبل از تصمیم قطعی، فرض خود را به‌صورت کامنت/TODO مشخص کن.
4. ویرایش عادی سند را نسخه جدید حساب نکن.
5. نسخه جدید فقط با عملیات `Add New Version` ایجاد شود.
6. سند یا نسخه قدیمی حذف فیزیکی نشود؛ فقط وضعیت قدیمی/آرشیوی بگیرد.
7. محتوای تأییدنشده سرپرست نباید برای اپراتور/کاربر عادی در UI یا API عمومی قابل مشاهده باشد.
8. مدیر می‌تواند مستقیم منتشر کند.
9. سرپرست برای سند رسمی، ویرایش و نسخه جدید نیازمند تأیید مدیر است.
10. پیام روزانه نیازمند تأیید نیست.
11. متن اصلی سند منبع اصلی محتواست. فایل‌ها پیوست هستند.
12. برای عملیات کلیدی Audit Log ثبت کن.
13. برای «مطالعه شد» KnowledgeReadLog ثبت کن.
14. API نرم‌افزار اصلی فقط محتوای مجاز و منتشرشده را برگرداند.

## روش توسعه پیشنهادی
- هر Feature را در یک Pull Request کوچک انجام بده.
- هر PR باید شامل migration، endpoint، validation، permission و تست پایه باشد.
- برای Frontend، هر صفحه باید loading/error/empty state داشته باشد.
- برای Backend، Swagger/OpenAPI را همگام نگه دار.

## Definition of Done برای AI
یک تسک وقتی کامل است که:
- کد کامپایل شود.
- migration یا data model کامل باشد.
- Permissionها رعایت شده باشند.
- Acceptance Criteria پاس شوند.
- Swagger یا API contract به‌روز باشد.
- خطاهای اصلی مدیریت شوند.
- تست پایه یا سناریوی تست دستی نوشته شود.
