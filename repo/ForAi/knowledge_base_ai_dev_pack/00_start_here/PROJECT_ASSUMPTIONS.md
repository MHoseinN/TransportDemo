# Project Assumptions

این فرض‌ها برای شروع توسعه هستند و در صورت تفاوت با تصمیم تیم، باید اصلاح شوند.

## دامنه
- پروژه فقط شامل ماژول پایگاه دانش است.
- سیستم تماس، دریافت تماس، ثبت تماس و اتصال تلفنی خارج از MVP فعلی است.
- نرم‌افزار اصلی از ماژول پایگاه دانش API می‌گیرد.

## نقش‌ها
- Manager / Admin: مدیر یا ادمین ارشد
- Supervisor: سرپرست
- Operator/User: اپراتور یا کاربر مشاهده‌کننده

## پشته فنی پیشنهادی
اگر پروژه موجود پشته مشخصی ندارد، این پشته پیشنهادی است:
- Backend: ASP.NET Core Web API
- Database: SQL Server
- ORM: Entity Framework Core
- Frontend: Vue 3 + TypeScript
- UI: Vuetify
- API Docs: Swagger/OpenAPI
- Auth: JWT یا اتصال به احراز هویت سازمانی

اگر تیم شما پشته متفاوتی دارد، فقط نام تکنولوژی‌ها را تغییر بده؛ قوانین محصولی ثابت می‌مانند.

## زبان UI
- زبان اصلی رابط کاربری فارسی و راست‌چین است.
- ویرایشگر متن باید RTL/LTR، اندازه متن و فونت‌های سازمانی رایج مثل B Nazanin و B Lotus را پشتیبانی کند.

## تاریخ و زمان
- تاریخ ثبت، صدور، انتشار، آخرین تغییر و تاریخ/ساعت ارسال پیام مهم است.
- تاریخ اعتبار سند اختیاری است.
