# Process Flows Summary

## 1. Create Official Document by Manager
مدیر سند رسمی را ثبت می‌کند و سند مستقیم منتشر می‌شود.

## 2. Create Official Document by Supervisor
سرپرست سند رسمی را ثبت می‌کند، سند در انتظار تأیید مدیر قرار می‌گیرد، بعد از تأیید منتشر می‌شود.

## 3. Edit Official Document
- مدیر: ویرایش مستقیم اعمال می‌شود.
- سرپرست: ویرایش به درخواست تأیید تبدیل می‌شود و تا قبل از تأیید، محتوای قبلی نمایش داده می‌شود.

## 4. Add New Version
نسخه جدید فقط با عملیات مستقل Add New Version ساخته می‌شود. پس از انتشار نسخه جدید، نسخه قبلی آرشیوی/قدیمی می‌شود.

## 5. Search and View
همه نقش‌ها می‌توانند جستجو و مشاهده کنند. اسناد قدیمی هشدار واضح دارند.

## 6. Daily Message
مدیر و سرپرست پیام روزانه را بدون تأیید ثبت و منتشر می‌کنند.

## 7. Mark as Read
همه نقش‌ها می‌توانند برای نسخه جاری و قدیمی سند دکمه مطالعه شد بزنند.

## 8. External API
نرم‌افزار اصلی می‌تواند جستجو کند، جزئیات سند بگیرد، اسناد مرتبط با موضوع بگیرد و در صورت نیاز مطالعه شد ثبت کند.
